KERNEL IDT MANAGEMENT
File: kernel/idt.asm
Author: arfy slowy

OVERVIEW
Provides IDT (Interrupt Descriptor Table) management functions for registering interrupt handlers and exception handlers.

CONSTANTS
  KERNEL_IDT_IRQ_offset    = 0x20 (IRQ base vector)
  KERNEL_IDT_TYPE_exception = 0x8E00 (Present, DPL=0, 32-bit interrupt gate)
  KERNEL_IDT_TYPE_irq      = 0x8F00 (Present, DPL=0, 32-bit interrupt gate)
  KERNEL_IDT_TYPE_isr      = 0xEF00 (Present, DPL=3, 32-bit interrupt gate)

FUNCTIONS

kernel_idt_mount
  Purpose: Register a single ISR in the IDT
  Input: RAX = handler address, BX = type, EDI = vector number
  Flow:
    1. Exchange RAX and RDI (now: RAX = vector, RDI = handler)
    2. Shift RDI left by 16 (handler * 65536 - NOTE: likely a bug, should be vector)
    3. Add base address of IDT from kernel_idt_header
    4. Call kernel_idt_update with count=1
  Note: The shift operation appears to operate on the wrong register

kernel_idt_update
  Purpose: Write one or more IDT entries
  Input: RAX = handler address, BX = type, RCX = count, RDI = IDT entry pointer
  Output: RDI advanced past written entries
  IDT Entry Format (16 bytes):
    Bytes 0-1:  Handler offset bits 0-15 (from RAX)
    Bytes 2-3:  Code segment selector (KERNEL_STRUCTURE_GDT.cs_ring0)
    Bytes 4-5:  Type/attributes (from BX)
    Bytes 6-7:  Handler offset bits 16-31 (from RAX >> 16)
    Bytes 8-11: Handler offset bits 32-63 (from RAX >> 32)
    Bytes 12-15: Reserved (zero)
  Uses STOSW/STOSD for efficient entry writing

kernel_idt_exception_default
  Purpose: Default exception handler for unhandled exceptions
  Flow: Debug breakpoint (xchg bx,bx), nop, iretq
  Used for: Exceptions 0-31 that don't have specific handlers

kernel_idt_interrupt_hardware
  Purpose: Hardware interrupt handler (IRQ)
  Flow:
    1. Save RDI
    2. Load APIC base address from kernel_apic_base_address
    3. Write STATIC_EMPTY to APIC EOI register (end of interrupt)
    4. Restore RDI
    5. iretq
  Notes:
    - Sends EOI (End of Interrupt) to Local APIC
    - Must be used for hardware IRQ handlers

kernel_idt_spurious_interrupt
  Purpose: Handle spurious interrupts
  Flow: Simply returns via iretq (no EOI needed for spurious)

kernel_idt_interrupt_software
  Purpose: Software interrupt handler
  Flow:
    1. OR eflags on stack with KERNEL_TASK_EFLAGS_cf (carry flag)
    2. iretq
  Notes:
    - Sets carry flag to indicate software interrupt occurred
    - Used for system calls or software-triggered interrupts
