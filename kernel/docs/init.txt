INITIALIZATION CHAIN
File: kernel/init.asm
Author: arfy slowy

OVERVIEW
Main initialization module that includes all init components in the correct order.
Handles transition from 32-bit to 64-bit mode, SMP application processor boot, and
complete kernel initialization.

STRUCTURE

SMP Boot Check (top of init.asm):
  1. Check kernel_init_smp_semaphore
     STATIC_FALSE -> BSP path (continue init)
     STATIC_TRUE  -> AP path, include ap.asm (application processor init)
     Falls through to .entry to continue boot

32-bit Pre-Initialization (before [BITS 64]):
  2. kernel/init/long_mode.asm
     Set up 64-bit long mode (GDT, paging, etc.)
     Switches CPU from 32-bit to 64-bit mode

  3. kernel/init/panic.asm
     Initialize panic handler for early errors

   4. kernel/init/data.asm
      Initialize data structures and strings
      Contains kernel_init_boot_file (incbin of boot.asm trampoline)
      Contains kernel_init_ap_semaphore for AP synchronization

   5. kernel/init/multiboot.asm
     Process Multiboot information from bootloader

[BITS 64] Transition:
  6. kernel/init/apic.asm
     Early APIC detection (before full init)
     Must run after 64-bit mode is enabled

64-bit Initialization (kernel_init):
  7. kernel/init/video.asm
     Initialize video output and display welcome message
     Calls: kernel_video_dump, kernel_video_string

  8. kernel/init/memory.asm
     Parse Multiboot memory map and initialize memory manager
     Uses: Multiboot info structure from bootloader
     Output: kernel_memory_map_address, page bitmap

  9. kernel/init/acpi.asm
     Initialize ACPI and enumerate CPUs
     Finds: RSDP, RSDT, MADT
     Output: kernel_apic_base_address, kernel_apic_count, IO-APIC info

   10. kernel/init/page.asm
      Set up paging and virtual memory mappings
      Allocates: PML4 table
      Maps: kernel, stack, video memory, APIC registers
      Maps: boot trampoline at 0x8000 for SMP AP wake

  11. kernel/init/gdt.asm
     Initialize Global Descriptor Table
     Sets up: null, CS, DS, TSS descriptors
     Loads: GDT and task register

  12. kernel/init/idt.asm
     Initialize Interrupt Descriptor Table
     Sets up: exception handlers (0-31), spurious handler
     Loads: IDT via LIDT, enables interrupts (sti)

  13. kernel/init/rtc.asm
     Initialize Real Time Clock
     Configures: 1024Hz periodic interrupts
     Connects: IRQ 8 to IO-APIC

   14. kernel/init/ps2.asm
      Initialize PS/2 keyboard controller
      Configures: scan code set, interrupt enable
      Note: Does NOT connect to IO-APIC (missing in code)

   15. kernel/init/ipc.asm
      Initialize IPC message pool
      Allocates: IPC pool page
      Must run before network and task init

   16. kernel/init/task.asm
      Initialize tasking system
      Creates: first kernel task
      Mounts: timer interrupt handler (IRQ 0)

   17. kernel/init/services.asm
      Start kernel services (shell, HTTP, network)

   18. kernel/init/smp.asm
      Copy boot trampoline (boot.asm) to 0x8000
      Set kernel_init_smp_semaphore to STATIC_TRUE
      Send INIT IPI to all APs (ICR = 0x00004500)
      Wait 10ms for APs to complete INIT
      Send STARTUP IPI with vector 0x08 to each AP (ICR = 0x00004608)
      APs start at boot.asm (16-bit real mode -> 32-bit -> kernel)

AP boot path (ap.asm):
  - Reload GDT, enable PAE, load PML4
  - Enable long mode (EFER.LME), enable paging
  - Far jump to 64-bit mode
  - Configure LAPIC: task priority, get APIC ID
  - Calculate and load per-CPU TSS selector
  - Load IDT
  - Wait on kernel_init_ap_semaphore
  - Allocate and map stack page via kernel_page_map_logical
  - Merge page tables with BSP via kernel_page_merge
  - Load merged CR3, set stack pointer
  - Release semaphore, call kernel_init_apic
  - Get APIC ID, find task slot, jump to kernel_task.ap_entry

Final Steps (after all init modules):
  - call kernel_init_apic (full APIC initialization)
  - Set APIC TICR register to DRIVER_RTC_Hz (1024Hz timer)
  - Send EOI to clear any pending interrupt
  - sti (enable interrupts)
  - smp.asm -> wake application processors

ERROR HANDLING
Each init module checks for errors and calls kernel_init_panic_* on failure.
