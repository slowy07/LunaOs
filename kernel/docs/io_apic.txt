IO-APIC SUPPORT
File: kernel/io_apic.asm
Author: arfy slowy

OVERVIEW
Provides functions to connect IRQ lines to the IO-APIC (Input/Output Advanced Programmable Interrupt Controller). The IO-APIC handles external device interrupts and routes them to the Local APIC.

CONSTANTS
  KERNEL_IO_APIC_ioregsel   = 0x00 (Register select)
  KERNEL_IO_APIC_iowin      = 0x10 (I/O window)
  KERNEL_IO_APIC_iowin_low  = 0x00 (Low dword of window)
  KERNEL_IO_APIC_iowin_high = 0x01 (High dword of window)
  KERNEL_IO_APIC_TRIGER_MODE_level = 0x8000 (Level-triggered mode)

VARIABLES
  kernel_io_apic_base_address = qword (IO-APIC MMIO base address)

FUNCTION

kernel_io_apic_connect
  Purpose: Connect an IRQ to the IO-APIC
  Input: EAX = IRQ vector, EBX = IO-APIC register offset
  Output: None
  
  The IO-APIC uses two I/O ports:
    - 0xFEC00000 + 0x00: Register select (ioregsel)
    - 0xFEC00000 + 0x10: I/O window (iowin)
  
  Flow:
    1. Add low offset to EBX
    2. Write register number to ioregsel
    3. Write low 32 bits of value to iowin
    4. Add high offset
    5. Write register number again
    6. Write high 32 bits of value to iowin

IO-APIC REDIRECTION ENTRY FORMAT
  Bits:
    0-7:   Interrupt vector
    8-10:  Delivery mode (0 = fixed)
    11:    Destination mode (0 = physical)
    12:    Polarity (0 = high active)
    13:    Trigger mode (0 = edge, 1 = level)
    14:    Mask (1 = masked)
    15:    Reserved
    16-23: Reserved
    24-55: Destination (APIC ID)
    56-63: Reserved

NOTES
The IO-APIC base address is discovered during ACPI initialization.
Each IRQ line has its own redirection entry in the IO-APIC.
