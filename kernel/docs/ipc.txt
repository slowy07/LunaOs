IPC - INTER-PROCESS COMMUNICATION
File: kernel/ipc.asm
Author: arfy slowy

OVERVIEW
Message-based IPC mechanism for communication between tasks.
Each message has a TTL (time-to-live) and is routed by destination PID.

STRUCTURES
KERNEL_IPC_STRUCTURE_LIST (80 bytes per entry):
  Offset  Size  Field
  0x00    8     ttl            - microsecond deadline, message expires when RTC > ttl
  0x08    8     pid_source     - sender task PID
  0x10    8     pid_destination - receiver task PID
  0x18    8     size           - size of attached data (0 = inline data)
  0x20    8     pointer        - pointer to external data block (if size > 0)
  0x28    32    data           - inline data buffer (used when size == 0)

CONSTANTS
KERNEL_IPC_SIZE_page_default = 1 (one 4KB page for the message pool)
KERNEL_IPC_ENTRY_limit = entries per page (4096 / 80)
KERNEL_IPC_TTL_default = ~10ms (RTC_Hz / 100)

GLOBAL VARIABLES
kernel_ipc_semaphore    - spinlock for concurrent access
kernel_ipc_base_address - pointer to IPC message pool (page-aligned)
kernel_ipc_entry_count  - number of active messages

PROCEDURES

kernel_ipc_insert:
  Inserts a message into the IPC pool.
  Input:  RBX = destination PID, RSI = data pointer (if RCX > 0)
          RCX = data size (0 for inline copy), stack = data for inline
  Output: CF = clean (success)
  Notes:  Acquires semaphore via macro_close.
          Searches for an expired or empty slot.
          If pool is full, follows linked list to next page.
          For inline data (RCX=0), copies into .data field.

kernel_ipc_receive:
  Retrieves a message addressed to the current task.
  Input:  none (uses active task PID)
  Output: CF = clean if found, set if no message
          RSI = pointer to data, RCX = data size (if pointer)
          Stack receives pointer + size for pointer-based data
  Notes:  Skips expired messages (TTL check).
          For pointer-based data, returns pointer and size on stack.
          For inline data, copies data to caller's stack.
