IPC - INTER-PROCESS COMMUNICATION
File: kernel/ipc.asm
Author: arfy slowy

OVERVIEW
Message-based IPC mechanism for communication between tasks.
Each message has a TTL (time-to-live) and is routed by destination PID.
The pool uses a linked-list of pages for linear growth.

STRUCTURES
KERNEL_IPC_STRUCTURE_LIST (72 bytes per entry):
  Offset  Size  Field
  0x00    8     ttl            - microsecond deadline, message expires when RTC > ttl
  0x08    8     pid_source     - sender task PID
  0x10    8     pid_destination - receiver task PID
  0x18    8     size           - size of attached data (reordered before .pointer)
  0x20    8     pointer        - pointer to external data block (if size > 0)
  0x28    32    other          - inline data buffer (was .data in prev version)

CONSTANTS
KERNEL_IPC_SIZE_page_default = 1 (one 4KB page for the message pool)
KERNEL_IPC_ENTRY_limit = entries per page (4096 / 80)
KERNEL_IPC_TTL_default = ~100ms (RTC_Hz / 10)

GLOBAL VARIABLES
kernel_ipc_semaphore    - spinlock for concurrent access
kernel_ipc_base_address - pointer to IPC message pool (page-aligned)
kernel_ipc_entry_count  - number of active messages (dq)

PROCEDURES

kernel_ipc_insert:
  Inserts a message into the IPC pool.
  Input:  RBX = destination PID, RSI = data pointer (if RCX > 0)
          RCX = data size (0 for inline copy), stack = data for inline
  Output: CF = clean (success), jump to .error if RBX = 0
  Notes:  Busy-waits if pool is full (kernel_ipc_entry_count >= limit).
          Acquires semaphore via macro_close before inserting.
          Searches for expired slot by comparing ttl to driver_rtc_microtime.
          TTL is set after data is written (not before).
          Inline data copied into .data field, pointer used if RCX > 0.
          When scanning reaches end of pool without finding a slot, jumps
          back to .loop to retry (instead of following linked-list pages).

kernel_ipc_receive:
  Retrieves a message addressed to the current task by copying the
  full KERNEL_IPC_STRUCTURE_LIST into a caller-provided buffer.
  Input:  RDI = pointer to caller's STRUCTURE_LIST buffer
  Output: CF = clean if found, set if no message
          Caller's buffer populated with the IPC entry data
  Notes:  Does not use semaphore (relies on single-consumer design).
          Early-exits with CF set if kernel_ipc_entry_count is STATIC_EMPTY.
          Skips non-matching pid_destination and expired messages (TTL check).
          Sets the found entry to STATIC_EMPTY (clears ttl) rather than
          maintaining a linked-list free pool.
          Decrements kernel_ipc_entry_count on success.
