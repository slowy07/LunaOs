KERNEL MAIN
File: kernel/kernel.asm
Author: arfy slowy

OVERVIEW
The main kernel file ties together all subsystems. After bootloader
transfers control to 0x100000, it runs initialization in 32-bit mode,
transitions to 64-bit long mode, completes hardware setup, then
executes `/bin/init` as the first userspace process. Error handling
after exec prints the error code and halts.

BOOT SEQUENCE DETAIL

Step 1: 32-bit entry at init:
  %include "kernel/init.asm"
  - Check SMP semaphore (BSP vs AP path)
  - Enable long mode (long_mode.asm -> 64-bit + paging)
  - Process Multiboot info, init panic handler
  - Load init data strings and structures

Step 2: 64-bit init (kernel_init label):
  - video.asm: Display welcome banner + runtime resolution
  - font.asm: Display font name
  - memory.asm: Parse memory map, init bitmap, display RAM count
  - acpi.asm: Find RSDP/RSDT or XSDP/XSDT, enumerate CPUs
  - page.asm: Set up PML4, map kernel/stack/video/APIC
  - gdt.asm: Null, CS, DS, TSS descriptors
  - idt.asm: Exceptions (0-31), syscall (0x40 -> kernel_service), spurious (0xFF)
  - rtc.asm: 1024Hz periodic timer, IRQ 8, sti
  - ps2.asm: Keyboard controller init
  - ipc.asm: IPC message pool allocation
  - vfs.asm: Root magicknot, /bin + /dev dirs, preload init/shell/wello
  - storage.asm: PCI IDE scan, reset, enumerate, VFS /dev/hd* knots
  - network.asm: PCI NIC detection, i82540EM init
  - task.asm: First kernel task, timer IRQ 0 mount
  - services.asm: Start tresher kernel service (tx, network, HTTP removed)

Step 3: Finalize:
  - call kernel_init_apic: Full APIC config, TICR = 1024Hz, EOI
  - smp.asm: Wake application processors via INIT/STARTUP IPI
  - Wait for all APs (kernel_init_ap_count == kernel_apic_count)
  - Set kernel_init_semaphore = STATIC_FALSE (signals services to start)

Step 4: kernel: - jump to /bin/init exec:
  mov ecx, kernel_init_exec_end - kernel_init_exec  ; length of "/bin/init"
  mov rsi, kernel_init_exec                          ; pointer to string
  call kernel_vfs_path_resolve                       ; find parent dir
  jc .error                                          ; error -> print code
  call kernel_vfs_file_find                          ; find file knot
  jc .error                                          ; error -> print code
  call kernel_exec                                   ; create ring-3 process
  jnc .end                                           ; success -> idle

.error: print error code (decimal) via kernel_video_number, then halt

Step 5: CPU halts in jmp $ (idle loop). If exec failed, error code
is printed in decimal first.

NOTE: The `clean:` step (release init code pages) and thread.asm
include are currently commented out.

CONFIGURATION CONSTANTS

KERNEL_BASE_address         0x0000000000100000  (1MB, kernel load address)
KERNEL_MEMORY_HIGH_REAL     0xFFFF800000000000  (high-half base)
KERNEL_STACK_address        KERNEL_MEMORY_HIGH_VIRTUAL - 2 pages
KERNEL_STACK_pointer        KERNEL_MEMORY_HIGH_VIRTUAL - 1 page
SOFTWARE_base_address       0xFFFF800000000000  (userspace code = high-half real)

kernel_init_exec            "/bin/init"          (first userspace process path)
