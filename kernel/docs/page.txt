PAGE MANAGEMENT
File: kernel/page.asm
Author: arfy slowy

OVERVIEW
Provides comprehensive page table management including:
- Page table allocation
- Physical memory mapping
- Virtual memory mapping
- Page frame allocation/deallocation

PAGE FLAGS
  KERNEL_PAGE_FLAG_available     = 0x01 (Present bit)
  KERNEL_PAGE_FLAG_write        = 0x02 (Writable bit)
  KERNEL_PAGE_FLAG_user         = 0x04 (User accessible)
  KERNEL_PAGE_FLAG_write_through = 0x08 (Write-through caching)
  KERNEL_PAGE_FLAG_cache_disable = 0x10 (Cache disabled)
  KERNEL_PAGE_FLAG_length       = 0x80 (1GB page / 2MB page)

PAGE TABLE SIZES
  KERNEL_PAGE_RECORDS_amount = 512 (entries per table)
  KERNEL_PAGE_PML1_SIZE_byte = 512 * 4096 = 2MB
  KERNEL_PAGE_PML2_SIZE_byte = 512 * 2MB = 1GB
  KERNEL_PAGE_PML3_SIZE_byte = 512 * 1GB = 512GB
  KERNEL_PAGE_PML4_SIZE_byte = 512 * 512GB = 256TB

VARIABLES
  kernel_page_pml4_address     = CR3 value (root page table)
  kernel_page_total_count      = Total pages available
  kernel_page_free_count      = Free pages
  kernel_page_reserved_count   = Reserved pages
  kernel_page_paged_count      = Pages used for page tables

FUNCTIONS

kernel_page_drain
  Purpose: Clear/drain a single page (4096 bytes)
  Input: RCX = size (IGNORED - always clears 4096 bytes)
  Output: Page at RDI is zeroed
  Notes:
    - RCX is NOT used - function always clears exactly one page
    - Calls internal .proceed with RCX = KERNEL_PAGE_SIZE_byte (4096)
    - ANDs RDI with KERNEL_PAGE_mask to align to page boundary

kernel_page_drain_few
  Purpose: Clear multiple pages
  Input: RCX = number of pages (NOT bytes)
  Flow: Shifts RCX left by 12 (multiply by 4096), then calls .proceed

kernel_page_map_physical
  Purpose: Identity map physical memory (virtual = physical)
  Input: RAX = physical address, BX = flags, RCX = number of pages (NOT bytes)
  Output: Carry set on error, clear on success
  Flow:
    1. Call kernel_page_prepare to set up page tables
    2. OR RAX with BX flags
    3. Loop: write PTE using STOSQ, increment RAX by 4096
    4. Check if R12 exceeds 512 (PML1 full), call kernel_page_pml1 if needed
    5. Decrement RCX (page counter), continue until zero
  Notes:
    - RCX is the PAGE COUNT, not byte size
    - Increments physical address by KERNEL_PAGE_SIZE_byte each iteration

kernel_page_map_logical
  Purpose: Map virtual address to physical memory
  Input: RAX = virtual address, BX = flags, RCX = number of pages (NOT bytes)
  Output: Carry set on error
  Flow:
    1. Call kernel_page_prepare
    2. For each page:
       - If PTE already exists, skip (use existing)
       - Otherwise allocate new physical page and create mapping
    3. Uses kernel_memory_alloc_page for new page allocations

kernel_page_prepare
  Purpose: Create page table hierarchy for address
  Input: RAX = address, BX = flags, R11 = PML4 base address
  Output: RDI = pointer to PML1 entry for address
  Modifies: R8-R15 (used for intermediate table pointers)
  Flow:
    1. Calculate PML4 index from bits 39-47
    2. Check if PML4 entry exists, allocate if needed
    3. Calculate PML3 index from bits 30-38
    4. Check if PML3 entry exists, allocate if needed
    5. Calculate PML2 index from bits 21-29
    6. Check if PML2 entry exists, allocate if needed
    7. Calculate PML1 index from bits 12-20
    8. Return RDI pointing to PML1 entry
  Notes:
    - Lazy allocation: only creates tables as needed
    - Each new table increments kernel_page_paged_count

kernel_page_pml1
  Purpose: Handle PML1 table full condition
  Called when R12 (PML1 entry index) reaches 512
  Creates new PML2 entry and PML1 table

kernel_page_merge
  Purpose: Recursively copy page table entries from source to destination
  Input: RSI = source PML*, RDI = destination PML*
  Recursive down to PML1 level

kernel_page_secure
  Purpose: Reserve pages (mark as non-free) for later use
  Input: RCX = number of pages to reserve
  Output: Carry set if not enough free pages
  Flow:
    1. Acquire memory lock
    2. Check if enough free pages available (free - reserved >= RCX)
    3. Subtract from free_count, add to reserved_count
    4. Release lock
