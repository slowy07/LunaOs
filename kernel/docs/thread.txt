THREAD EXECUTION
File: kernel/thread.asm
Author: arfy slowy

OVERVIEW
Provides kernel_thread for creating threads within the current task's
address space. Threads share CR3 (page table) with their parent task.
Note: currently commented out in kernel/kernel.asm (testing exec/service).

PROCEDURE

kernel_thread:
  Creates a new thread that executes in the same address space as the caller.
  Input:  RSI = thread entry point
          R8  = pointer to caller's task structure (for CR3)
          Stack = initial RSP value
  Output: CF set on allocation failure
  Flow:
    1. Allocate a page for the thread stack and drain page table
    2. Map KERNEL_STACK_address to the allocated page
    3. Build iretq frame on stack: RIP, CS, EFLAGS, RSP, DS
    4. Set RSP in task structure to point to the frame
    5. Merge thread's stack page into parent's page table
    6. Call kernel_task_add with KERNEL_TASK_FLAG_thread flag
