RTC (Real Time Clock) driver provides hardware timekeeping functionality for
the LunaOS kernel. It interfaces with the CMOS RTC chip via I/O ports and
utilizes IRQ 8 for periodic interrupts.

The driver maintains:
  - Current time (seconds, minutes, hours)
  - Current date (day, month, year, day of week)
  - Microtime counter (incremented on each RTC interrupt)


HARDWARE INTERFACE

I/O Ports Used:
  +---------------------+------------+
  | Port                | Purpose    |
  +---------------------+------------+
  | 0x0070 (command)    | CMOS addr  |
  | 0x0071 (data)       | CMOS data  |
  +---------------------+------------+

CMOS Registers (accessed via port 0x70):
  +-------+------------------+----------------------------------------+
  | Addr  | Register         | Description                            |
  +-------+------------------+----------------------------------------+
  | 0x00  | Seconds          | Current second (0-59)                 |
  | 0x02  | Minutes          | Current minute (0-59)                 |
  | 0x04  | Hours            | Current hour (0-23)                   |
  | 0x06  | Weekday          | Day of week (1-7)                     |
  | 0x07  | Day of Month     | Day (1-31)                            |
  | 0x08  | Month            | Month (1-12)                          |
  | 0x09  | Year             | Year (0-99)                           |
  | 0x0A  | Status Reg A     | Update/in-progress flag, rate config |
  | 0x0B  | Status Reg B     | Mode flags (24h, binary, etc)        |
  | 0x0C  | Status Reg C     | Interrupt flags                       |
  +-------+------------------+----------------------------------------+

Status Register A Bits:
  +-----------------------------------+--------+
  | Bit                               | Value  |
  +-----------------------------------+--------+
  | Update In Progress (UIP)          | 0x80   |
  | Divider                           | 0x20   |
  | Rate (default 1024Hz = 0x06)      | 0x06   |
  +-----------------------------------+--------+

Status Register B Bits:
  +-----------------------+--------+
  | Flag                  | Value  |
  +-----------------------+--------+
  | Daylight Savings      | 0x01   |
  | 24-Hour Mode          | 0x02   |
  | Binary Mode           | 0x04   |
  | Periodic Interrupt    | 0x40   |
  +-----------------------+--------+

DATA STRUCTURES

Time Structure (DRIVER_RTC_STRUCTURE_TIME):
  Offset    Size    Field           Description

  0x00      1       second          Seconds (BCD format)
  0x01      1       minute          Minutes (BCD format)
  0x02      1       hour            Hours (BCD format)
  0x03      1       day             Day of month (BCD format)
  0x04      1       month           Month (BCD format)
  0x05      1       year            Year (BCD format)
  0x06      1       day_of_week    Day of week (BCD format)

Global Variables:
  +-----------------------+--------+--------------------------------------+
  | Variable              | Type   | Description                          |
  +-----------------------+--------+--------------------------------------+
  | driver_rtc_semaphore  | byte   | Lock for thread-safe access         |
  | driver_rtc_microtime  | byte*  | Interrupt counter (time base)       |
  | driver_rtc_time       | byte*  | Cached time values                  |
  +-----------------------+--------+--------------------------------------+

  * NOTE: Both driver_rtc_microtime and driver_rtc_time are declared as
    single bytes (db STATIC_EMPTY) but are accessed as larger structures:
    - driver_rtc_microtime: inc qword [driver_rtc_microtime]
    - driver_rtc_time: accessed via offsets up to +6 (7 bytes total)
    
    This may cause memory corruption and should be fixed by declaring them
    as appropriate structures (qword for microtime, 7-byte buffer for time).

FUNCTIONS

1. driver_rtc (ISR - Interrupt Service Routine)
   Purpose: Handle RTC periodic interrupts (IRQ 8)
   
   Flow:
   
   +------------------+
   | Start ISR        |
   +--------+---------+
            |
            v
   +------------------+
   | Breakpoint (DBG) |
   +--------+---------+
            |
            v
   +------------------+
   | Save RAX         |
   +--------+---------+
            |
            v
   +------------------+
   | Increment        |  <-- microtime counter
   | microtime        |
   +--------+---------+
            |
            v
   +------------------+
   | Read RTC data    |  <-- Clears interrupt flag
   | port (0x71)      |
   +--------+---------+
            |
            v
   +------------------+
   | Send EOI to APIC |
   | (EOI register)   |
   +--------+---------+
            |
            v
   +------------------+
   | Restore RAX      |
   +--------+---------+
            |
            v
   +------------------+
   | iretq            |  <-- Return from interrupt
   +------------------+

   Note: Reading from data port (0x71) automatically clears the interrupt
         request flag in the RTC hardware.

2. driver_rtc_get_date_and_time
   Purpose: Read current date and time from RTC
   
   Flow:
   
   +--------------------+
   | Start function     |
   +--------+-----------+
            |
            v
   +--------------------+
   | Save RAX           |
   +--------+-----------+
            |
            v
   +--------------------+
   | Write 0x00 (sec)   | --> Command port 0x70
   | to command port    |
   +--------+-----------+
            |
            v
   +--------------------+
   | Read seconds       | <-- Data port 0x71
   | from data port     |
   +--------+-----------+
            |
            v
   +--------------------+
   | Store in buffer    | --> driver_rtc_time.second
   +--------+-----------+
            |
            v
   +--------------------+
   | Write 0x02 (min)  | --> Command port 0x70
   | to command port    |
   +--------+-----------+
            |
            v
   +--------------------+
   | Read minutes       | <-- Data port 0x71
   | from data port     |
   +--------+-----------+
            |
            v
   +--------------------+
   | Store in buffer    | --> driver_rtc_time.hour
   +--------+-----------+
            |
            v
   +--------------------+
   | Restore RAX        |
   +--------+-----------+
            |
            v
   +--------------------+
   | Return             |
   +--------------------+
   


INTERRUPT CONFIGURATION

IRQ Line: 8 (mapped to vector 0x08)
APIC Register: KERNEL_IO_APIC_iowin + (0x08 * 0x02) = 0x20 + 0x10 = 0x30

The periodic interrupt frequency is configured to 1024 Hz by default
(DRIVER_RTC_Hz equ 1024), which corresponds to rate bits 0x06 in Status Reg A.

The interrupt:
  - Fires every ~1ms (1024 times per second)
  - Increments microtime counter
  - Clears pending interrupt by reading data port

USAGE NOTES

1. Time Format:
   - Values are stored in BCD (Binary Coded Decimal) format
   - Example: 0x25 means 25, not 37 decimal

2. Thread Safety:
   - driver_rtc_semaphore should be acquired before reading time
   - The ISR increments microtime without taking the semaphore (interrupt
     context)

3. Initialization Sequence:
   a. Set IRQ 8 mapping in IO/APIC
   b. Configure Status Register A with desired rate (0x06 for 1024Hz)
   c. Configure Status Register B for 24-hour mode, binary mode if desired
   d. Enable periodic interrupts in Status Register B (0x40)

4. Known Issues:
   - driver_rtc_get_date_and_time stores minutes into the hour field
   - Only seconds and minutes/hours are read; day/month/year not implemented
