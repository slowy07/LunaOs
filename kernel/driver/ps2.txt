PS/2 CONTROLLER DRIVER
File: kernel/driver/ps2.asm

OVERVIEW
Provides driver support for PS/2 keyboard and mouse devices. The PS/2 controller
is the standard interface for these devices on x86 systems.

HARDWARE INTERFACE

I/O Ports:
+------------------------------+------------+--------------------------------+
| Port                         | Access     | Description                    |
+------------------------------+------------+--------------------------------+
| 0x064 (STATUS/COMMAND)      | Read/Write | Status register (read)         |
|                              |            | Command register (write)       |
| 0x060 (DATA)                | Read/Write | Data port                      |
+------------------------------+------------+--------------------------------+

Status Register Bits:
+------+--------------------------------------------+
| Bit  | Description                                |
+------+--------------------------------------------+
| 0    | Output buffer full (read)                 |
| 1    | Input buffer full (write)                 |
| 2    | System flag (self-test passed)            |
| 3    | Command/data (0=data, 1=command)          |
| 5    | Second port output buffer full            |
| 6    | Timeout error                            |
| 7    | Parity error                             |
+------+--------------------------------------------+

INTERRUPTS

IRQ Assignments:
+------------------+--------------+--------------------------------+
| Device           | IRQ Number   | IO-APIC Register              |
+------------------+--------------+--------------------------------+
| Keyboard         | 0x01         | 0x20 + (1 * 2) = 0x22        |
| Mouse            | 0x0C         | 0x20 + (12 * 2) = 0x34      |
+------------------+--------------+--------------------------------+

PS/2 CONTROLLER COMMANDS

Command Register (0x64):
+------------------+------------------------------------------------+
| Command          | Description                                    |
+------------------+------------------------------------------------+
| 0x20             | Read configuration byte                        |
| 0x60             | Write configuration byte                      |
| 0xA7             | Disable second port                           |
| 0xA8             | Enable second port                            |
| 0xA9             | Test second port                              |
| 0xAA             | Test controller                               |
| 0xAB             | Test first port                              |
| 0xAD             | Disable first port                           |
| 0xAE             | Enable first port                            |
| 0xD0             | Read controller output port                   |
| 0xD4             | Write to second port                          |
+------------------+------------------------------------------------+

DEVICE COMMANDS (via 0x60)

+------------------+------------------------------------------------+
| Command          | Description                                    |
+------------------+------------------------------------------------+
| 0xF2             | Get device ID                                 |
| 0xF3             | Set sample rate                               |
| 0xF4             | Enable packets                                |
| 0xF5             | Disable scanning                              |
| 0xF6             | Set default values                            |
| 0xFF             | Reset device                                  |
+------------------+------------------------------------------------+

DEVICE RESPONSES

+------------------+------------------------------------------------+
| Response         | Description                                    |
+------------------+------------------------------------------------+
| 0xAA             | Self test passed                              |
| 0xFA             | Command acknowledged                          |
| 0xFC             | Error                                         |
| 0xFE             | Resend request                               |
+------------------+------------------------------------------------+

FUNCTIONS

driver_ps2_keyboard
  Purpose: Keyboard interrupt handler (IRQ 1)
  Flow:
    1. Read scan code from port 0x60
    2. Check if it's a sequence prefix (0xE0 or 0xE1)
    3. If prefix, store and return (wait for next byte)
    4. If sequence active, combine prefix + scan code
    5. Look up in current matrix (low or high)
    6. Handle key release (>= 0x80)
    7. Call driver_ps2_keyboard_shift to handle shift/capslock
    8. Call driver_ps2_keyboard_save to add to cache
    9. Send EOI to APIC
  Notes:
    - Prefix 0xE0 stored, next byte combined with AH=0xE0
    - Release code = make code + 0x80

driver_ps2_keyboard_shift
  Purpose: Handle shift and capslock state changes
  Input: AX = translated key code
  Flow:
    - Checks for shift left/right press/release
    - Checks for capslock press/release
    - On capslock release, clears semaphore (not toggled)
    - Calls driver_ps2_keyboard_matrix_change when state changes

driver_ps2_keyboard_save
  Purpose: Save key to keyboard cache ring buffer
  Input: AX = key code
  Flow:
    1. Shift cache left by 16 bits (make room)
    2. Store new key in lowest 16 bits
  Notes:
    - Cache is 64-bit, holds 4 x 16-bit keys
    - Oldest key at bits 48-63, newest at bits 0-15

driver_ps2_keyboard_matrix_change
  Purpose: Toggle between normal/shift keyboard matrix
  Flow:
    1. Swap driver_ps2_keyboard_matrix pointer
    2. If it was low, switch to high
    3. If it was high, switch to low
  Notes:
    - Called when shift/capslock state changes

driver_ps2_keyboard_read
  Purpose: Read a key from the keyboard cache
  Output: AX = key code (0 if cache empty), ZF set if empty
  Flow:
    1. Read key from bits 32-47 (oldest position)
    2. Shift cache left by 16 bits
    3. Test if AX is zero (empty slot)

driver_ps2_check_write
  Purpose: Wait for input buffer to be empty
  Flow: Loop reading status port until bit 1 is clear

driver_ps2_check_read
  Purpose: Wait for output buffer to be full
  Flow: Loop reading status port until bit 0 is set

driver_ps2_send_command
  Purpose: Send command to PS/2 controller
  Input: AL = command byte
  Flow:
    1. Wait for input buffer empty
    2. Write command to port 0x64

driver_ps2_send_command_receive_answer
  Purpose: Send command and receive response
  Input: AL = command byte
  Output: AL = response byte
  Flow:
    1. Send command via driver_ps2_send_command
    2. Read response via driver_ps2_receive_answer

driver_ps2_receive_answer
  Purpose: Receive response from PS/2 device
  Output: AL = response byte
  Flow:
    1. Wait for output buffer full
    2. Read data from port 0x60

driver_ps2_send_answer_or_ask_device
  Purpose: Send data/command to PS/2 device
  Input: AL = data byte
  Flow:
    1. Wait for input buffer empty
    2. Write data to port 0x60

driver_ps2_check_dummy_answer_or_dump
  Purpose: Clear any pending data in output buffer
  Flow:
    1. Loop: check status port bit 0
    2. If data available, read and discard
    3. Continue until buffer empty

KEYBOARD CACHE

The keyboard uses a 4-entry ring buffer to store recent key events.

Structure (64-bit):
  Bits 0-15   : Oldest key
  Bits 16-31  : Second key
  Bits 32-47  : Third key
  Bits 48-63  : Fourth key (newest)

Reading:
  driver_ps2_keyboard_read extracts the oldest key from the cache
  Cache is then shifted left (newest moves to oldest position)

NOTES
  - Standard PS/2 keyboard uses scan code set 2
  - Mouse support is present but not fully implemented
  - Both ports can be used, but this driver uses only port 1 (keyboard)
