This module performs the critical transition from 32-bit protected mode to
64-bit long mode (x86-64). It sets up the initial page tables for identity
mapping in 64-bit mode and enables the necessary CPU features.

This code runs BEFORE the kernel is relocated to its final virtual address.

Page Table Setup:

  | Constant                          | Value
  | KERNEL_INIT_LONG_MODE_PML4_TABLE_address | 0x0000A000
  | KERNEL_INIT_LONG_MODE_PAGE_SIZE_4KiB_byte | 0x1000 (4 KB)
  | KERNEL_INIT_LONG_MODE_PAGE_SIZE_2MiB_byte | 0x00200000 (2 MB)

Page Flags:
  | Flag                              | Value
  | KERNEL_INIT_LONG_MODE_PAGE_FLAG_available   | 0x01 (Present)
  | KERNEL_INIT_LONG_MODE_PAGE_FLAG_writeable   | 0x02 (Writable)
  | KERNEL_INIT_LONG_MODE_PAGE_FLAG_2MiB_size   | 0x80 (2MB page)

PAGE TABLE LAYOUT
The initialization creates a minimal 4-level page table structure:

  PML4 (Page Map Level 4) @ 0x0000A000
  +-------+---------------------------+
  | Entry | Maps to                   |
  +-------+---------------------------+
  | 0     | PDPT @ 0x0000B000 (self) |
  | 1     | PDPT @ 0x0000C000         |
  +-------+---------------------------+

  PDPT (Page Directory Pointer Table) @ 0x0000B000
  +-------+---------------------------+
  | Entry | Maps to                   |
  +-------+---------------------------+
  | 0     | PD @ 0x0000D000           |
  +-------+---------------------------+

  PD (Page Directory) @ 0x0000D000 (2MB pages)
  +-------+---------------------------+
  | Entry | Physical Address         |
  +-------+---------------------------+
  | 0     | 0x0000000000200000        |
  | 1     | 0x0000000000400000        |
  | ...   | ...                       |
  | 511   | 0x000000003FE00000        |
  +-------+---------------------------+

This creates identity mapping for first 1GB of physical memory.

INITIALIZATION FLOW
1. CLEAR PAGE TABLES
   - Zero out 24KB (6 pages × 4KB) of memory at 0x0000A000
   - Use REP STOSD for fast zeroing

2. CONFIGURE PML4
   - PML4[0] → PDPT @ 0x0000B000 (identity)
   - PML4[1] → PDPT @ 0x0000C000 (for higher half)

3. CONFIGURE PDPT
   - PDPT[0] → PD @ 0x0000D000

4. CONFIGURE PD (2MB PAGES)
   - Create 512 entries covering first 1GB
   - Each entry maps 2MB of physical memory
   - Flags: Present + Writable

5. LOAD GDT
   - Load 64-bit GDT with code and data segments

6. ENABLE PAE
   - Set CR4.PAE (bit 5) = 1

7. LOAD PML4
   - Set CR3 = 0x0000A000

8. ENABLE LONG MODE
   - Read MSR 0xC0000080 (EFER)
   - Set EFER.LME (bit 8) = 1

9. ENABLE PAGING
   - Set CR0.PG (bit 31) = 1
   - Set CR0.WP (bit 16) = 1

10. JUMP TO 64-BIT MODE
    - Far jump to selector 0x0008 (CS Ring 0)
    - This clears CS pipeline and enters 64-bit mode

GDT SETUP (64-bit)

| Entry                  | Binary           | Description
| Null                   | 0x0000000000000000| Required null
| Code Segment (CS)      | 0x0020980000000000| 64-bit code, Present, R0
| Data Segment (DS)      | 0x0020920000000000| 64-bit data, Present, R0

Segment Selector:
  - Selector 0x0008: Offset 0x08 in GDT (CS)
  - Selector 0x0010: Offset 0x10 in GDT (DS)
