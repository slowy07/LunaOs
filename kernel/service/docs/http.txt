HTTP SERVICE
File: kernel/service/http.asm
Author: arfy slowy

OVERVIEW
Minimal HTTP/1.0 web server. Listens on port 80, responds to "GET /"
requests with a 200 OK HTML page, returns 404 for other paths.

CONSTANTS
SERVICE_HTTP_version  = "0"
SERVICE_HTTP_revision = "8"

MACROS
service_http_macro_foot:
  Emits HTML footer with version info and CSS styling.

PROCEDURES

service_http:
  Main service loop:
  1. Allocate service_http_ipc_message buffer
  2. Assign port 80 via service_network_tcp_port_assign
  3. Loop: receive IPC via kernel_ipc_receive (blocks until message arrives)
  4. Extract data pointer from KERNEL_IPC_STRUCTURE_LIST.other
  5. Compare request to "GET / "
  6. If match, send 200 OK via service_network_tcp_port_send
  7. If no match, send 404 Not Found
  8. Debug breakpoint (xchg bx, bx) before answer
  9. Release received page via kernel_memory_release_page
  10. Repeat from step 3

DATA
service_http_get_root:    "GET / " (comparison string)
service_http_200_default: HTTP/1.0 200 OK + HTML body
service_http_404:         HTTP/1.0 404 Not Found + HTML body

DATA
service_http_ipc_message  - KERNEL_IPC_STRUCTURE_LIST.SIZE buffer for IPC receive

NOTES
- Uses service_network_tcp_port_* functions (refactored from kernel_network)
- Now loops continuously via IPC receive (was single-request before)
- Service lifecycle managed by the init scheduler
- HTML includes inline CSS with green/red/white color scheme
- All responses include LunaOS version footer via macro
