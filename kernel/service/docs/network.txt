NETWORK SERVICE
File: kernel/service/network.asm
Author: arfy slowy

OVERVIEW
Implements a basic IPv4/TCP/UDP/ICMP/ARP network stack as a service task.
Runs as an independent task in the service list (kernel_init_services_list).
Communicates with the NIC driver and TX service entirely via IPC
(kernel_ipc_insert / kernel_ipc_receive). Loops continuously: receives
IPC messages from the NIC driver, processes each packet, forwards to TX
service for transmission via IPC.

CONSTANTS
SERVICE_NETWORK_RECEIVE_PUSH_WAIT_limit = 3 max spin iterations on receive
SERVICE_NETWORK_MAC_mask = 0x0000FFFFFFFFFFFF
SERVICE_NETWORK_STACK_SIZE_page = 1 page for TCP connection table
SERVICE_NETWORK_PORT_SIZE_page = 1 page for port table (up to 512 ports)
SERVICE_NETWORK_PORT_FLAG_empty, received, send - port state flags
SERVICE_NETWORK_STACK_FLAG_busy - TCP connection busy marker

SERVICE_NETWORK_FRAME_ARP_OPCODE_answer = 0x0200

STRUCTURES
SERVICE_NETWORK_STRUCTURE_MAC (6 bytes)
SERVICE_NETWORK_STRUCTURE_FRAME_ETHERNET (14 bytes): target, source, type
SERVICE_NETWORK_STRUCTURE_FRAME_ARP (28 bytes)
SERVICE_NETWORK_STRUCTURE_FRAME_IP (20 bytes): version, length, addresses, etc.
SERVICE_NETWORK_STRUCTURE_FRAME_ICMP (8+32=40 bytes): type, code, checksum, reserved, data
SERVICE_NETWORK_STRUCTURE_FRAME_UDP (8 bytes): ports, length, checksum
SERVICE_NETWORK_STRUCTURE_FRAME_TCP (20 bytes): ports, seq, ack, flags, etc.
SERVICE_NETWORK_STRUCTURE_FRAME_TCP_PSEUDO_HEADER (12 bytes): for checksum
SERVICE_NETWORK_STRUCTURE_TCP_STACK (48 bytes): per-connection state
SERVICE_NETWORK_STRUCTURE_PORT (8 bytes): per-port registration (pid)

GLOBAL VARIABLES
service_network_pid          - PID of this service (self-registered at start)
service_network_rx_count     - received packet counter
service_network_tx_count     - transmitted packet counter
service_network_port_semaphore - spinlock for port table access
service_network_port_table   - pointer to port table page
service_network_stack_address - pointer to TCP connection table
service_network_ipc_message  - KERNEL_IPC_STRUCTURE_LIST.SIZE buffer for IPC receive

PROCEDURES

service_network:
  Main loop. Registers PID via kernel_task_active, then loops:
  1. Receive IPC message via kernel_ipc_receive into service_network_ipc_message
  2. Extract packet pointer from IPC message
  3. Dispatch to ARP or IP handler based on Ethernet type
  4. Release packet page and repeat

service_network_transfer:
  Sends a packet to the TX service via IPC.
  Input:  EAX = packet length, RDI = packet data address
  Uses service_tx_pid as the destination PID for kernel_ipc_insert.
  Replaces direct calls to service_tx_add (now removed).

service_network_ip:
  IP dispatcher. Routes based on protocol field to ICMP/TCP handlers.

service_network_arp:
  Handles ARP requests. Responds with ARP reply containing own MAC address.
  Skips non-Ethernet/IPv4/mismatched target IP. Forwards via
  service_network_transfer.

service_network_icmp:
  Handles ICMP echo requests (ping). Computes IP header length, copies ICMP
  data payload from request to reply (32 bytes), computes checksums, and
  queues ICMP echo reply via service_network_transfer.

service_network_tcp:
  TCP state machine dispatcher. Routes to SYN/ACK/FIN/PSH-ACK handlers.
  Validates port number (< 512) and port table entry.
  Port table offset: port * SERVICE_NETWORK_STRUCTURE_PORT.SIZE.

service_network_tcp_syn:
  Handles incoming SYN. Allocates TCP connection slot, stores remote
  MAC/IP/ports, generates initial sequence number, sends SYN-ACK via
  service_network_transfer.

service_network_tcp_ack:
  Handles pure ACK. Clears pending ACK flag, checks for FIN+ACK completion.
  No longer contains debug breakpoint.

service_network_tcp_fin:
  Handles FIN. Updates sequence numbers, sends ACK+FIN via
  service_network_transfer, notifies connection end.

service_network_tcp_psh_ack:
  Handles PSH+ACK (data push). Copies TCP payload data, pads
  remaining page with zeros, then sends data to the registered
  port owner via kernel_ipc_insert using the PID from the port table.

service_network_tcp_find:
  Searches TCP connection table for matching MAC/IP/port pair.
  Returns connection structure pointer and IP header length.
  Input: RSI = Ethernet frame
  Output: CF set if not found, RDI = connection, RBX = IP header length

service_network_tcp_port_send:
  Sends data on an existing TCP connection.
  Allocates a page, builds TCP segment with PSH+ACK flags, wraps in
  IP/Ethernet, queues via service_network_transfer.

service_network_tcp_wrap:
  Builds TCP segment from connection state. Fills ports, seq, ack, flags,
  window, computes checksum via pseudo-header, calls IP wrapper.

service_network_ip_wrap:
  Builds IP header from connection state. Sets version, length, ID, TTL,
  protocol, computes checksum, calls Ethernet wrapper.

service_network_tcp_pseudo_header:
  Computes TCP pseudo-header checksum contribution.

service_network_tcp_port_assign:
  Registers a port for the current task (PID stored in port entry).
  Uses service_network_port_semaphore for exclusive access.
  Input: CX = port number (< 512)
  Output: CF set if port >= 512 or port already in use

service_network_tcp_port_receive:
  Checks port for received data. Returns data pointer/size if available.
  Input: CX = port number
  Output: CF set if no data, RBX = connection, RCX = size, RSI = pointer

service_network_ethernet_wrap:
  Wraps a frame with Ethernet header (target MAC, source MAC, type).

service_network_checksum:
  Standard internet checksum (ones' complement).
  Accumulates 16-bit words, folds carry, returns complement.

service_network_checksum_part:
  Partial checksum accumulator (for pseudo-header chaining).

NOTES
- NIC driver forwards received packets to this service via kernel_ipc_insert
  using service_network_pid as the target.
- This service forwards outgoing packets to the TX service via
  service_network_transfer (kernel_ipc_insert using service_tx_pid).
- Port table entries are keyed by PID (task identifier) for isolation.
- TCP PSH+ACK data is forwarded to the registered port owner's task
  via kernel_ipc_insert (direct IPC, not polling).
