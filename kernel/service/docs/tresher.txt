TASK RESOURCE RECLAIM SERVICE
File: kernel/service/tresher.asm
Author: arfy slowy

OVERVIEW
Background service that finds closed tasks and releases their resources
(memory pages, page table entries). The name "tresher" is short for
"task resource reaper."

PROCEDURES

service_tresher:
  Main loop:
  1. Call service_tresher_search to find a closed task
  2. Release task stack pages via kernel_memory_release_foreign
  3. If task is a thread, release its 4 PML4 entries
     (256 * 8 bytes each for recursive mapping)
  4. Purge the task's page table via kernel_page_purge
  5. Release the task structure page
  6. Decrement kernel_page_paged_count
  7. Clear task flags, loop back

service_tresher_search:
  Scans task list for a task with KERNEL_TASK_FLAG_closed set.
  Walks the linked task pages (KERNEL_STRUCTURE_TASK).
  Output: RSI = task structure with closed flag

NOTES
- Only closed tasks (not daemon or active) are reclaimed
- Thread tasks skip the full PML4 teardown (share parent's CR3)
- kernel_page_paged_count tracks total allocated page count
