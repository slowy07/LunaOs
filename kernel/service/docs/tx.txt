TRANSMIT SERVICE
File: kernel/service/tx.asm
Author: arfy slowy

OVERVIEW
Receives IPC messages containing packets to transmit. Checks the sender
PID: if the message comes from the network service (service_network_pid),
transmits the packet via driver_nic_i82540em_transfer; otherwise, frees
the page without transmitting. Runs as a standalone service task with its
own PID registered in service_tx_pid.

STRUCTURES
SERVICE_TX_STRUCTURE_CACHE (16 bytes):
  Offset  Size  Field
  0x00    8     size   (packet length)
  0x08    8     address (packet data pointer)

GLOBAL VARIABLES
service_tx_pid            - PID of the TX service (self-registered at start)
service_tx_ipc_message    - KERNEL_IPC_STRUCTURE_LIST.SIZE buffer for IPC receive

PROCEDURES

service_tx:
  Main loop:
  1. Register PID via kernel_task_active, store in service_tx_pid
  2. Loop: receive IPC message via kernel_ipc_receive into
     service_tx_ipc_message
  3. If no size, skip (empty message)
  4. Check pid_source: if from service_network_pid, call
     driver_nic_i82540em_transfer
  5. Otherwise, free the packet page via kernel_memory_release
  6. Repeat

NOTES
- Replaced the old cache-based design (service_tx_cache_*,
  service_tx_semaphore, service_tx_add) with pure IPC.
- The old cache queue with descending pointer was removed entirely.
- service_tx_pid is used by service_network_transfer to send packets
  for transmission.
- Allocated pages are freed after transmit via library_page_from_size
  + kernel_memory_release.
