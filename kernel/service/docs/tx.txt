TRANSMIT SERVICE
File: kernel/service/tx.asm
Author: arfy slowy

OVERVIEW
Receives IPC messages containing packets to transmit. Always sends all
received packets via driver_nic_i82540em_transfer, bypassing the old
PID source check. Runs as a standalone service task with its own PID
registered in service_tx_pid.

GLOBAL VARIABLES
service_tx_pid            - PID of the TX service (self-registered at start)
service_tx_ipc_message    - KERNEL_IPC_STRUCTURE_LIST.SIZE buffer for IPC receive

PROCEDURES

service_tx:
  Main loop:
  1. Register PID via kernel_task_active, store in service_tx_pid
  2. Loop: receive IPC message via kernel_ipc_receive into
     service_tx_ipc_message
  3. If no size, skip (empty message)
  4. jmp .send (bypass PID check, always transmit)
  5. Otherwise, free the packet page via kernel_memory_release
  6. Repeat

NOTES
- Removed SERVICE_TX_CACHE_SIZE_page and SERVICE_TX_STRUCTURE_CACHE.
- Now unconditionally sends packets (removed the PID source filter that
  previously only accepted packets from service_network_pid).
- service_tx_pid is used by service_network_transfer to send packets
  for transmission.
- Allocated pages are freed after transmit via library_page_from_size
  + kernel_memory_release.
